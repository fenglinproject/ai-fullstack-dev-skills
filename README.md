# AI 单人全栈开发 Skills 超细分版

一个面向 **单人全栈开发者 / 独立开发者 / AI 辅助编程用户** 的开发流程 Skills 工具包。

它把完整的软件开发流程拆成可复用的 AI Skills，覆盖：

```text
项目想法 → 需求澄清 → MVP → PRD → 页面设计 → 架构设计
→ 数据库设计 → 接口设计 → 前后端开发 → 联调 → 测试
→ 安全检查 → 部署上线 → 运维迭代
```

你只需要告诉 AI：“我要做一个什么项目”，AI 就可以根据当前阶段自动选择对应 Skill，按真实开发流程一步一步协助你推进。

---

## 为什么做这个项目？

一个人做全栈项目时，最大的问题往往不是不会写代码，而是缺少完整团队角色：

- 没有产品经理，需求容易不清楚
- 没有架构师，数据库和接口容易设计混乱
- 没有测试人员，异常场景容易漏掉
- 没有安全人员，容易忽略越权、SQL 注入、XSS、Token、文件上传等风险
- 没有运维人员，部署、回滚、备份、日志监控不规范
- 使用 AI 写代码时，AI 经常直接写代码，缺少完整软件工程流程

所以这个项目的目标是：

> 让一个人开发项目时，也能拥有接近完整团队的开发流程。

---

## 解决了什么问题？

### 1. 让 AI 不只是写代码

很多 AI 编程失败，不是代码不会写，而是前期没有做好：

- 需求分析
- MVP 控制
- PRD 文档
- 数据库设计
- 接口设计
- 权限设计
- 测试用例
- 安全检查
- 部署方案

本项目让 AI 按流程推进，而不是一上来直接写代码。

---

### 2. 帮单人开发者补齐团队能力

AI 会根据 Skills 临时扮演：

- 产品经理
- 架构师
- 前端工程师
- 后端工程师
- 测试工程师
- 安全工程师
- 运维工程师
- 项目经理

---

### 3. 降低返工和上线风险

通过需求评审、异常场景、接口规范、安全检查、部署清单等 Skills，减少：

- 需求反复变动
- 数据库重改
- 接口前后不一致
- 权限漏洞
- 上线后 Bug
- 部署不可回滚

## 快速使用

### 1. 先复制总控提示词

打开：

```text
00_MASTER_PROMPT.md
```

复制里面的内容给 AI，让 AI 进入“单人全栈开发总控官”模式。

---

### 2. 告诉 AI 你要做什么项目

例如：

```text
我要做一个校园二手交易小程序。
```

AI 会自动进入需求澄清阶段，先帮你确认用户、功能、MVP、业务流程和异常场景。

---

### 3. 按阶段继续推进

你可以直接说：

```text
继续生成 PRD
```

```text
根据 PRD 设计数据库
```

```text
生成接口文档
```

```text
开始写登录模块后端
```

```text
帮我做项目安全检查
```

```text
准备部署上线
```

AI 会根据当前任务自动使用对应 Skill。

## 注意事项

- 本项目是 AI 辅助开发流程工具，不是自动生成完整项目的脚手架。
- AI 输出的方案和代码需要开发者自行 review。
- 涉及支付、隐私、权限、安全的功能，必须人工检查。
- 不建议跳过需求、数据库、接口设计直接写代码。
- 上线前建议执行安全检查和部署检查。
